# Core Web Vitals 기준 정리 (요약)

> **목표**: 서비스 성능을 “괜찮다”고 말할 수 있는 최소 기준을 한눈에 보기 위함.  
> 아래 값들은 **Google이 공식 문서에서 제시한 기준**을 정리한 것이다.

---

## 1. LCP (Largest Contentful Paint)

- **정의**: 화면에서 가장 큰 콘텐츠(히어로 이미지, 큰 텍스트 등)가 그려지는 데 걸리는 시간.
- **단위**: ms (밀리초)

| 구간       | 값 (ms 기준)         | 의미          |
| ---------- | -------------------- | ------------- |
| **좋음**   | **≤ 2,500ms (2.5초)** | 사용자 체감 양호 |
| 개선 필요  | 2,500ms ~ 4,000ms    | 로딩이 다소 느림 |
| 나쁨       | **> 4,000ms (4초 초과)** | 많이 느린 편     |

---

## 2. INP (Interaction to Next Paint)

- **정의**: 유저가 클릭/탭/키 입력을 했을 때, **화면이 반응해서 다음 페인트가 일어날 때까지** 걸리는 시간.
- **단위**: ms (밀리초)

| 구간       | 값 (ms 기준)           | 의미              |
| ---------- | ---------------------- | ----------------- |
| **좋음**   | **≤ 200ms**            | 즉각적인 반응 느낌 |
| 개선 필요  | 200ms ~ 500ms          | 살짝 굼뜬 느낌     |
| 나쁨       | **> 500ms (0.5초 초과)** | 많이 버벅거림       |

---

## 3. CLS (Cumulative Layout Shift)

- **정의**: 화면 요소가 갑자기 밀리거나 내려가서 **레이아웃이 튀는 정도**를 점수로 누적한 값.
- **단위**: 점수 (0 이상, 1 이상은 거의 최악)

| 구간       | 값 (점수 기준) | 의미                     |
| ---------- | -------------- | ------------------------ |
| **좋음**   | **≤ 0.1**      | 눈에 거슬리는 튐 거의 없음 |
| 개선 필요  | 0.1 ~ 0.25     | 가끔 거슬리는 수준         |
| 나쁨       | **> 0.25**     | 자주/심하게 튐            |

---

## 4. TTFB (Time To First Byte) [참고용]

- **정의**: 브라우저가 서버로 요청을 보낸 뒤, **첫 바이트를 받기까지 걸리는 시간**.
- **단위**: ms (밀리초)

| 구간       | 값 (ms 기준)   | 의미                  |
| ---------- | -------------- | --------------------- |
| **좋음**   | **≤ 800ms**    | 서버 응답 양호          |
| 개선 필요  | 800ms ~ 1,800ms | 다소 느림(백엔드/네트워크 점검) |
| 나쁨       | **> 1,800ms**  | 서버/네트워크 병목 의심   |

---

## 5. “괜찮다”의 최소 기준 (초기 목표)

- LCP: **2.5초 이하**
- INP: **200ms 이하**
- CLS: **0.1 이하**
- TTFB: **800ms 이하** (참고)
